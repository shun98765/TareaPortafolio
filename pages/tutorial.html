¿Qué es un archivo .gitignore?

Imagina que tienes una caja de juguetes. Dentro de esta caja, tienes muchos juguetes, pero solo quieres mostrar algunos a tus amigos. Los juguetes que no quieres mostrar, los guardas en una caja aparte para que no estén a la vista.

En el mundo de las computadoras, cuando estás trabajando en un proyecto con un programa llamado Git, a veces tienes archivos en tu proyecto que no quieres compartir con otras personas o que no quieres que se mezclen con los archivos importantes. Aquí es donde entra el archivo .gitignore.

¿Para qué sirve?

El archivo .gitignore le dice a Git qué archivos o carpetas debe ignorar (no mirar ni compartir). Es como una lista de cosas que Git no debe poner en la caja de juguetes que enseñas a tus amigos.

¿Cómo se usa?

Vamos a ver cómo hacer y usar un archivo .gitignore paso a paso:

1. Crear un archivo .gitignore

  1. Abre tu computadora y ve al lugar donde está tu proyecto. Imagina que es tu cuarto donde tienes todos tus juguetes.
  2. Crea un nuevo archivo y nómbralo .gitignore.

2. Añadir archivos y carpetas a ignorar

  1. Abre el archivo .gitignore que acabas de crear. Imagina que es una lista en la que vas a escribir qué juguetes no quieres mostrar.
  2. Escribe el nombre de los archivos o carpetas que quieres ignorar. Por ejemplo, si no quieres que Git mire una carpeta llamada juguetes_sucios, escribe juguetes_sucios/.

Ejemplos de cosas que se podrían añadir:

  • *.txt para ignorar todos los archivos que terminan en .txt.
  • imagen_*.png para ignorar todos los archivos que empiezan con imagen_ y terminan en .png.
  • carpeta_secreta/ para ignorar una carpeta llamada carpeta_secreta.

3. Guardar el archivo

  1. Guarda el archivo .gitignore en el mismo lugar donde está tu proyecto. Es como guardar la lista de juguetes que no quieres mostrar en un lugar especial.

4. Verificar que funciona

  1. Haz algunos cambios en tu proyecto y añade archivos que deberían ser ignorados según tu lista. Por ejemplo, pon algunos archivos en la carpeta juguetes_sucios.
  2. Mira cómo Git está manejando estos archivos. Git debería ignorar todos los archivos y carpetas que has puesto en tu archivo .gitignore.

¡Y eso es todo! Ahora sabes cómo funciona y cómo usar el archivo .gitignore. Es como hacer una lista para asegurarte de que Git solo se enfoque en lo que realmente quieres mostrar a tus amigos.
